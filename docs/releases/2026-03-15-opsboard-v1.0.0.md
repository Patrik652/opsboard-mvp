# Opsboard v1.0.0

Opsboard v1.0.0 promotes the project from a styled MVP demo to a portfolio-grade production workspace.

## Highlights

- Added explicit session runtime modes for `demo` and `authenticated`
- Added Google sign-in flow and authenticated app gate
- Added per-user Firestore data model and repository abstraction
- Added idempotent starter workspace bootstrap for first login
- Wired boards, incidents, status, analytics, operations, AI, and audit pages to live workspace data
- Added deterministic copilot over incidents, cards, services, and audit history
- Added Firestore security rules for `users/{uid}/**`
- Added end-to-end workflow coverage for demo workspace, incidents, and audit trail

## Product Outcomes

- Demo mode now works as a fast evaluation path without requiring auth
- Authenticated mode persists real user data in Firestore
- Incident changes generate audit events automatically
- Status, analytics, and operations derive from the same workspace snapshot
- Copilot recommendations are deterministic, testable, and explainable
